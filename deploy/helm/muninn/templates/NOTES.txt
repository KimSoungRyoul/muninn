▶ Huginn Operator
  배포 확인:
    kubectl -n {{ .Release.Namespace }} rollout status deploy/{{ include "muninn.operator.fullname" . }}
    kubectl -n {{ .Release.Namespace }} get pods -l app.kubernetes.io/component=operator
  CRD 확인:
    kubectl get crds | grep muninn.io
  {{- if not .Values.operator.webhook.enabled }}

    --set operator.webhook.enabled=true
  {{- else }}
  webhook: 활성(cert-manager 필요). caBundle 주입까지 수십 초 걸릴 수 있습니다.
  {{- end }}
  {{- if .Values.operator.metrics.serviceMonitor.enabled }}
  ⚠️ ServiceMonitor 활성: Prometheus Operator(monitoring.coreos.com CRD)가 클러스터에 설치돼 있어야 합니다.
  {{- end }}
{{- end }}

{{- if .Values.web.enabled }}

▶ Muninn Web (콘솔 + Muninn API + 메모리)
  접속:
    kubectl -n {{ .Release.Namespace }} port-forward svc/{{ include "muninn.web.fullname" . }} {{ .Values.web.service.port }}:{{ .Values.web.service.port }}
    open http://localhost:{{ .Values.web.service.port }}
  {{- if not .Values.web.auth.existingSecret }}
  ⚠️ CopilotKit 자격 미설정. 코파일럿을 쓰려면 Secret 생성 후 --set web.auth.existingSecret=muninn-web-secrets:
    kubectl -n {{ .Release.Namespace }} create secret generic muninn-web-secrets \
      --from-literal=claude-code-oauth-token="$CLAUDE_CODE_OAUTH_TOKEN"
  {{- end }}
  워크스페이스(멀티테넌시 §2) = MUNINN_WORKSPACE = {{ .Values.web.workspace | default .Release.Namespace }} (요청 헤더 x-muninn-workspace 로 override).
  {{- if .Values.web.oidc.issuer }}
  ✅ OIDC 활성(issuer={{ .Values.web.oidc.issuer }}): 운영자 콘솔 경로(승인/거절/위임)는 SSO JWT 를 JWKS 로 검증합니다.
  {{- else }}
  운영자 콘솔 SSO 를 쓰려면 OIDC 를 활성화하세요(미설정 시 정적 토큰만 사용):
    --set web.oidc.issuer=https://<idp>/realms/muninn --set web.oidc.audience=muninn-web
  {{- end }}
{{- end }}

{{- if and .Values.operator.enabled .Values.agent.serviceAccount.create }}

▶ Agent 인증 (operator 가 만드는 Job + muninn-web 이 공유)
  자격 Secret({{ .Values.agent.secrets.name }})은 보안상 chart 가 만들지 않습니다. 런타임에 생성:
    kubectl -n {{ .Release.Namespace }} create secret generic {{ .Values.agent.secrets.name }} \
      --from-literal=claude-code-oauth-token="$CLAUDE_CODE_OAUTH_TOKEN" \
      --from-literal=muninn-api-token="$(openssl rand -hex 32)"
  muninn-api-token 키를 넣으면 muninn-web 의 상태변경 API(보고·승인·위임·webhook)가 Bearer 인증을 강제하고,
  operator 가 agent Job 에도 같은 토큰을 주입합니다(미설정 시 web 은 dev 모드로 인증 비활성+경고).
{{- end }}

  이 chart 는 PostgreSQL 을 설치하지 않습니다(외부 의존성). 권장: CloudNativePG(CNPG) provision.
  {{- if .Values.metaDb.enabled }}
  muninn-web 에 DATABASE_URL 이 Secret/{{ .Values.metaDb.existingSecret }} (key={{ .Values.metaDb.uriKey }}) 에서 주입됩니다.
  {{- else }}
  metaDb.enabled=true + metaDb.existingSecret 로 muninn-web 의 메모리 DB 를 연결하세요.
  CNPG quickstart: deploy/quickstart/README.md (검색은 텍스트 전용 — pgvector 불필요).
  {{- end }}
